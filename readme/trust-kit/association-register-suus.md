## 1. Introduction

{% hint style="warning" %}
An Association is a legal entity, created and governed by its members, that provides the neutral rules and\
structures needed to enable trusted data sharing between companies.
{% endhint %}

The Association Register (ASR) is an important component in the BDI architecture. It is a centralized component that enables efficient data sharing between independent companies by providing a neutral member-governed entity that serves as a trust anchor. BDI connectors query the ASR to establish a trusted data exchange between participants.&#x20;

Note that, without an ASR, trust is negotiated bilaterally. The ASR introduces standardized trust, which reduces costs and risks, thereby enabling faster collaboration,  trust is standardized, which reduces cost and risk.&#x20;

### 1.1. Purpose

The ASR provides at least the following services:

* registration of information about participants,
* local certificate authority that issues BDI Signing Certificates,
* issuer of signed Assurance tokens,
* maintenance of synced trust lists,
* event notification to members/users.

### 1.2 Core functions

The core functions of ASRs are given below.&#x20;

<details>

<summary>Trust Anchor</summary>

The ASR serves as a "trust anchor" for identity, authorization and compliance of its members.&#x20;

</details>

<details>

<summary>Rules</summary>

The ASR defines and enforces common rules regarding onboarding, periodic checks and assessments and dispute resolution.&#x20;

</details>

<details>

<summary>Neutral-member driven governance</summary>

The ASR provides **neutral**-member driven governance. Neutrality means that no single party dominates.&#x20;

</details>

<details>

<summary>Operating members, systems, credentials, and endpoints</summary>

The ASR operates an association register of members, systems, credentials, and&#x20;endpoints.

</details>

<details>

<summary>Relationships</summary>

The ASR records system‚Äìorganization relationships.

</details>

<details>

<summary>Onboarding support</summary>

The ASR offers onboarding support, and operational monitoring of the association&#x20;components.

</details>

<details>

<summary>Dispute handling</summary>

The ASR facilitates dispute handling and arbitration between participants.&#x20;

</details>

<details>

<summary>Collaboration between associations</summary>

The ASR allows scalable collaboration/interoperability with other associations.&#x20;

</details>

<details>

<summary>Additional service</summary>

The ASR provides optional service integration, helpdesk, backoffice services and usage&#x20;accounting and settlement services.

</details>
